# Studentia Consent API Reference

Base URL (local): `http://localhost:3000`

Auth:
- Currently auth is disabled in backend middleware for local development.
- If re-enabled later, send: `Authorization: Bearer <API_TOKEN>`.

## 1) Health
### Request
`GET /health`

### Example response
```json
{
  "ok": true
}
```

## 2) Grant consent
### Request
`POST /consents/grant`

Body:
```json
{
  "studentId": "student-001",
  "receiverGroup": "Recruiters",
  "dataGroup": "Portfolio",
  "dataBlob": "optional payload"
}
```

### Example response
```json
{
  "ok": true,
  "txId": "KQ6...ABC",
  "returnValue": "GRANTED:student-001:Recruiters:Portfolio"
}
```

## 3) Revoke consent

## 5) On-chain status for one tuple
### Request
`GET /consents/onchain/:studentId/:receiverGroup/:dataGroup`

### Example
`GET /consents/onchain/student-001/Recruiters/Portfolio`

### Example response
```json
{
  "ok": true,
  "boxKey": "student-001:Recruiters:Portfolio",
  "status": "granted",
  "numeric": 1
}
```

Status meanings:
- `granted` → `numeric: 1`
- `revoked` → `numeric: 0`
- `none` → box missing

## 7b) Upload document via multipart/form-data
### Request
`POST /documents/upload-form`

Form fields:
- `studentId` (text)
- `receiverGroup` (text)
- `dataGroup` (text)
- `file` (file)

### cURL example
```bash
curl -X POST http://localhost:3000/documents/upload-form \
  -F "studentId=student-001" \
  -F "receiverGroup=Recruiters" \
  -F "dataGroup=Portfolio" \
  -F "file=@./resume.pdf"
```

    "receiverGroup": "Recruiters",

    "sizeBytes": 23456,
    "storageMode": "encrypted",
    "sharedWith": [],
    "createdAt": "2026-02-20T..."

}

## 17) Create custom data group
### Request
`POST /data-groups`

Body:
```json
{
  "studentId": "student-001",
  "name": "Research"
}
```

### Example response
```json
{
  "ok": true,
  "dataGroup": {
    "id": "67ca...",
    "studentId": "student-001",
    "name": "Research",
    "isCustom": true,
    "createdAt": "2026-02-20T..."
  }
}
```

Notes:
- Default groups (`Academics`, `Portfolio`, `Personal`) are always available.
- Upload APIs validate `dataGroup` against default + student custom groups.
- Invalid upload group returns:
```json
{
  "error": "invalid dataGroup for this student. Create custom groups via POST /data-groups first"
}
```

## 24) Optional requester identity check during download
`GET /documents/download/:id?ownerStudentId=student-001&requesterGroup=Recruiters&requesterId=67ef...`

Notes:
- `requesterId` is optional to preserve compatibility with existing clients.
- When provided, backend validates that requester is an active member of the given request group.

## Algorand note (real execution)
Consent grant/revoke and consent checks use live Algorand RPC calls against the configured network (`ALGOD_SERVER`, `APP_ID`, signer mnemonic). This is not simulated behavior.

## Troubleshooting
- `Invalid API Token`: set backend `.env` `ALGOD_TOKEN` to LocalNet value.
- `invalid Box reference`: backend/frontend must send the same tuple values used for the box key.
- `Unable to connect`: backend not running on port 3000.
